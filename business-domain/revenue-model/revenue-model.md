# Revenue Models Comparison: SaaS vs License vs Hybrid vs Source Code

## 1. Overview

This document compares four common software revenue models:

- SaaS (Software as a Service)
- License-only (On-premise)
- Hybrid (License + Service)
- Source Code Model (New ⭐)

It explains:
- how revenue is generated
- how cost is structured
- how to estimate pricing (feature + human hour)

---

# 2. SaaS Model

## 2.1 What you sell
Access to a system hosted by you (cloud-based)

## 2.2 Revenue Model

- Per user (e.g. $10/user/month)
- Per usage
- Tiered pricing

### Example

100 users × $20/month × 12  
= **$24,000/year**

---

## 2.3 Cost Structure

### Fixed
- Development team

### Variable
- Cloud
- Support
- Scaling

---

## 2.4 Cost Estimation (Human Hour + Feature)

Example feature breakdown:

| Feature | Estimated Hours |
|--------|--------------|
| User login | 16h |
| Product module | 80h |
| Reporting | 60h |

Total = 156h

Cost:
156h × $50/hour = **$7,800**

---

## 2.5 Profit Logic

- Scale-driven
- High upfront cost
- High long-term margin

# 6. Key Comparison

| Model | Revenue | Cost Driver | Risk |

# 7. Core Cost Estimation Method ⭐

## 👨‍💻 Feature + Human Hour Model

### Step 1: Break into features
- login
- product
- reporting

### Step 2: Estimate hours per feature

### Step 3: Apply rate

Formula:

Cost =  
Σ (feature hours) × hourly rate

---

### Example

| Feature | Hours |
|--------|------|
| Login | 20h |
| Product | 100h |
| Report | 80h |

Total = 200h

Rate = $50/h

Cost =  
200 × 50 = **$10,000**
